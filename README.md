# 카드 리포트 (여자친구용)

카드 소비 실적 분석 + 카드별 최적 결제 판정 + 전월실적 트래킹 + 스샷 OCR 일괄입력 웹앱.
[card-report-me](../card-report-me)와 완전히 별도의 앱/데이터임 (같은 기기에서 둘 다 설치해도 데이터 안 섞임).

## GitHub Pages로 배포하기

1. 이 폴더 내용을 새 저장소에 push (파일명 `index.html` 그대로 유지 - 루트에 있어야 함)
   ```bash
   git init
   git add .
   git commit -m "init"
   git branch -M main
   git remote add origin <레포 URL>
   git push -u origin main
   ```
2. GitHub 저장소 → Settings → Pages → Source를 "Deploy from a branch" → Branch: `main` / `/(root)` 선택 → Save
3. 몇 분 뒤 `https://<username>.github.io/<repo명>/` 로 접속 가능

## iPhone에 앱처럼 설치하기

1. 위 URL을 Safari로 열기
2. 공유 버튼 → "홈 화면에 추가"
3. 홈 화면 아이콘으로 실행 (일반 앱처럼 전체화면으로 뜸)

## 데이터 저장 방식

- 모든 결제 데이터는 브라우저 `localStorage`에 저장됨 (기기 로컬, 서버로 전송 안 됨)
- GitHub Pages 도메인으로 접속해야 데이터가 안정적으로 유지됨
- 데이터 백업/이동은 앱 내 "설정" 탭 → Export/Import 기능 사용

## 스샷 자동 인식(OCR) 기능

- "일괄" 탭에서 카드사 앱 이용내역 스샷을 올리면 [Tesseract.js](https://github.com/naptha/tesseract.js)(jsDelivr CDN)로 텍스트를 자동 인식함
- 처음 실행 시 한글 인식 데이터를 내려받느라 다소 걸림 (인터넷 연결 필요)
- 인식 결과는 항상 미리보기에서 확인/수정 후 저장

## 카드 혜택 데이터(`card_db.json`) 업데이트하기

`index.html` 안의 `CARD_DB` 객체가 실제 계산 로직에 쓰이는 원본임. 혜택 변경 시 이 부분을 직접 수정.

## 현재 등록된 카드 (3)

| 카드 | 연회비 | 전월실적 | 상태 |
|---|---|---|---|
| 네이버 현대카드 (오리지널) | 10,000원 | 30만원 | ⚠️ 단종 (재발급 불가) |
| 하나 제이드 클래식 | 120,000원 | 50만원 | 발급중 |
| 하나 네이버페이 체크카드 | 0원 | 25만원 | ⚠️ 신규발급 중지 |

## 알려진 한계 / 주의사항

- 네이버 현대카드(오리지널)와 하나 네이버페이 체크카드는 **재발급이 안 되는 카드**라서, 앱 상 연회비 대비 효율이 낮게 나와도 해지는 신중하게 판단해야 함 (앱 "추천" 탭에 경고 문구 있음)
- 하나 제이드 클래식은 연 소비 600만원(월 50만원) 기준이 프리미엄 바우처/공항라운지 유지 여부를 가르는 핵심 조건 - 이 앱은 그 부분까지는 자동 판정하지 않으니 별도로 체크 필요
- 할인율 계산은 여러 조건(전월실적 미충족 시 기본적립만 적용되는 로직 등)을 단순화한 근사치임
